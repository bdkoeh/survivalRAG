  <name>Task 1: Create generation module with init, config, and system prompts</name>
  <files>pipeline/generate.py</files>
  <action>
Create pipeline/generate.py with:

1. Module docstring explaining purpose: LLM response generation via Ollama with streaming, mode-specific prompts, and safe defaults for medical/survival content.

2. Imports: os, logging, re (for later use), typing (Iterator, Optional), ollama.

3. Module-level state (matching retrieve.py pattern):
   - `_model: str = ""`
   - `_validated: bool = False`

4. Constants:
   - `DEFAULT_MODEL = "llama3.2:3b"` -- 2GB, 128K context, good instruction-following
   - `_SAFE_OPTIONS` dict with locked medical/safety defaults:
     - temperature: 0.2
     - top_p: 0.85
     - top_k: 20
     - repeat_penalty: 1.1
     - num_ctx: 8192 (prevents silent context truncation per Pitfall 2 from research)
   - `_MODE_OPTIONS` dict mapping mode -> num_predict overrides:
     - full: 1024
     - compact: 512
     - ultra: 80

5. Three system prompt constants (each mode gets its own -- per locked decision, single LLM call per query, no post-processing summarization):

   SYSTEM_PROMPT_FULL:
   - "You are a survival and emergency preparedness reference tool."
   - "Answer ONLY using the reference context provided. Do not use your own knowledge."
   - FORMAT RULES: cite sources as (Source: <document name>, p.<page>), number steps for procedures, bullets for lists, bold all safety warnings with **WARNING:**, start with safety warnings BEFORE answer, never diagnose
   - Explicit: "You are a reference tool, NOT a medical provider. Never diagnose conditions."

   SYSTEM_PROMPT_COMPACT:
   - Brief version: max 3-4 short paragraphs, cite sources, bold warnings, never diagnose
   - "Answer ONLY from the provided context."

   SYSTEM_PROMPT_ULTRA:
   - Under 200 characters, telegram style (short phrases, no articles)
   - Include critical safety warnings, no citations (no room)
   - Never diagnose

6. `init(model: str = None) -> None`:
   - Set `_model` from argument, else `SURVIVALRAG_MODEL` env var, else `DEFAULT_MODEL`
   - Validate model via `ollama.show(_model)`:
     - Catch `ollama.ResponseError`: raise RuntimeError with "Model '{_model}' is not available. Run: ollama pull {_model}"
     - Catch generic Exception with "connection"/"refused" in str: raise ConnectionError with "Ollama is not running. Start it with: ollama serve"
   - Set `_validated = True`
   - Log model name at INFO level

7. `_get_system_prompt(mode: str) -> str`:
   - Return the appropriate system prompt constant for mode ("full", "compact", "ultra")
   - Raise ValueError for unknown mode

8. `_get_options(mode: str) -> dict`:
   - Start with copy of `_SAFE_OPTIONS`
   - Override num_predict from `_MODE_OPTIONS[mode]`
   - Return options dict

Important: Do NOT import from pipeline.prompt in this file yet -- that's Plan 05-02's integration task.

  <files>pipeline/generate.py</files>
  <action>
Add to the existing pipeline/generate.py created in Task 1:

1. `generate_stream(prompt: str, mode: str = "full") -> Iterator[str]`:
   - Check `_validated` -- raise RuntimeError("Generation engine not initialized. Call init() first.") if False
   - Get system prompt via `_get_system_prompt(mode)`
   - Get options via `_get_options(mode)`
   - Call `ollama.generate(model=_model, prompt=prompt, system=system, options=options, stream=True)`
   - Iterate over chunks, yielding `chunk.response` (handle both attribute and dict access: `chunk.response if hasattr(chunk, 'response') else chunk.get("response", "")`)
   - Only yield non-empty tokens
   - Log at DEBUG level: mode and model being used

   IMPORTANT per Anti-Pattern from research: Use `ollama.generate()` NOT `ollama.chat()`. The prompt from assemble_prompt() is a pre-assembled string. Pass system prompt separately via the `system` parameter and the assembled context+query as the `prompt` parameter.

2. `generate(prompt: str, chunks: list[dict], mode: str = "full") -> dict`:
   - Collect all tokens from `generate_stream(prompt, mode=mode)` into a list
   - Join tokens into `response_text`
   - Return structured dict (this is the Phase 5 -> Phase 6/7 contract):
     ```python
     {
         "response": response_text,
         "mode": mode,
         "model": _model,
         "verification": None,  # Populated by Plan 05-02
     }
     ```
   - NOTE: Post-processing and citation verification will be added by Plan 05-02. This task returns raw LLM output in the response dict. The `chunks` parameter is accepted but not used yet -- it's needed for the verification step in Plan 05-02.
   - Log response length and mode at INFO level

Per locked decision: generate_stream() is the streaming interface (CLI prints directly, web UI wraps in SSE), generate() is the full-response interface. Both provided per CONTEXT.md.
