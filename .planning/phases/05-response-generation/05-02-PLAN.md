  <name>Task 1: Add citation verification and post-processing functions</name>
  <files>pipeline/generate.py</files>
  <action>
Add the following functions to pipeline/generate.py (after the existing generate() function):

1. Add `import re` and `from difflib import SequenceMatcher` to imports (re may already be imported from Task 1).

2. `extract_citations(response_text: str) -> list[str]`:
   - Extract citation references from LLM response text using multiple regex patterns to handle format variation (per Pitfall 3 from research: small LLMs produce inconsistent formats):
     - `r'\(Source:\s*([^,\)]+)'` -- (Source: FM 21-76, ...)
     - `r'\[([^\]]+?)(?:,\s*p\.?\s*\d+)?\]'` -- [FM 21-76, p.45]
     - `r'\(([A-Z][A-Z\s\-\d\.]+\d+)\)'` -- (FM 21-76)
     - `r'(?:per|from|in)\s+([A-Z][A-Z\s\-\d\.]+\d+)'` -- per FM 21-76
   - Normalize: strip whitespace, deduplicate via set
   - Return list of unique citation strings

3. `verify_citations(response_text: str, retrieved_chunks: list[dict], threshold: float = 0.6) -> dict`:
   - Extract citations via extract_citations()
   - If no citations found: return `{"passed": True, "citations_found": 0, "citations_verified": 0, "citations_failed": 0, "details": [], "note": "No citations found in response"}`
   - Collect unique source_document values from chunk metadata
   - For each citation, match against source docs:
     - Fast path: substring match (citation.lower() in source.lower() or vice versa) -> score 1.0
     - Slow path: `SequenceMatcher(None, citation.lower(), source.lower()).ratio()`
   - Citation verified if best_score >= threshold
   - Return structured dict:
     ```python
     {
         "passed": bool,  # True if ALL citations verified
         "citations_found": int,
         "citations_verified": int,
         "citations_failed": int,
         "details": [{"citation": str, "matched_source": str|None, "score": float, "verified": bool}, ...]
     }
     ```
   - Per locked decision: verification results are BOTH logged AND returned as structured data for Phase 6 evaluation framework

4. `_post_process(text: str, mode: str = "full") -> str`:
   - CRITICAL: Never modify content words, dosages, or measurements -- strictly structural formatting only
   - Ultra mode: if len(text) > 220, truncate at last sentence boundary (. or !) before 200 chars. If no boundary found after char 100, truncate at 200 + "..."
   - Full/compact modes:
     - Ensure WARNING/CAUTION/DANGER lines are bold: `re.sub(r'(?m)^(WARNING|CAUTION|DANGER):\s*', r'**\1:** ', text)`
     - Ensure numbered steps have consistent formatting: `re.sub(r'(?m)^(\d+)[.\)]\s*', r'\1. ', text)`
   - Return processed text

5. Update the existing `generate()` function to wire in post-processing and verification:
   - After collecting tokens and joining into response_text:
     - Apply `_post_process(response_text, mode=mode)`
     - Run `verify_citations(response_text, chunks)` -- UNLESS mode == "ultra" (ultra strips citations, so verification is skipped with `{"passed": True, "skipped": True, "reason": "ultra mode strips citations"}`)
     - If verification failed (not passed and not skipped): append "\n\nNote: Some citations could not be verified against source documents." to response_text
     - Return updated dict with verification result populated (replace the None from Plan 05-01)

Per locked decisions:
- Post-generation verification runs AFTER full response is complete, NOT during streaming (Pitfall 4)
- Fuzzy matching handles abbreviations and partial titles
- On verification failure: keep the response, append visible warning
