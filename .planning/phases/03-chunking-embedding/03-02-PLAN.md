  <action>
**Create pipeline/benchmark.py** -- Automated embedding benchmark that generates query-document pairs from the actual corpus and evaluates Recall@5.

**Prerequisites check at script start:**
- Verify Ollama is running and nomic-embed-text model is available (via `get_model_version()`)
- Verify at least one LLM model is available for query generation (try `llama3.1:8b` first, then `llama3.1`, then any available model from `ollama.list()`)
- Verify `processed/sections/` directory exists and has content

Key constants:
- `MIN_PAIRS = 50` -- minimum query-document pairs required
- `TARGET_PAIRS = 60` -- target to exceed minimum comfortably
- `RECALL_K = 5` -- Recall@K metric
- `PASS_THRESHOLD = 0.85` -- 85% Recall@5 required
- `PAIRS_PATH = Path("processed/benchmark/pairs.jsonl")`
- `RESULTS_PATH = Path("processed/benchmark/results.json")`

**Phase 1: Sample chunks from corpus**

`_sample_chunks_from_corpus(n: int = TARGET_PAIRS) -> list[dict]`:
- Scan all `processed/sections/*/` directories
- For each section file: call `read_section_file()` to get metadata + content
- Filter to sections with content length >= 100 characters (skip near-empty sections)
- Stratified sampling strategy:
  - Group sections by document (source_document field)
  - Sample proportionally from each document to get diversity
  - Aim for `n` samples total
- For each sampled section: call `chunk_section()` to produce chunks, take the first chunk (or a random chunk if multiple)
- Return list of dicts: `{"chunk_text": str, "source_document": str, "section_header": str, "content_type": str, "categories": list}`

**Phase 2: Auto-generate queries**

`_generate_query(chunk_text: str, query_type: str, llm_model: str) -> str`:
- Use `ollama.chat()` with the available LLM to generate a realistic query for the chunk
- Three query types (distribute evenly across pairs):
  1. `"lay_language"` -- Prompt: "Generate a search query a non-expert would type to find this information. Use simple everyday language, no technical terms."
  2. `"medical_terminology"` -- Prompt: "Generate a search query using proper medical or military terminology for this information."
  3. `"typo_variant"` -- Prompt: "Generate a search query with a common typo or misspelling that someone might type quickly in an emergency."
- Truncate chunk_text to first 500 chars in the prompt to keep LLM context small
- Parse response: strip whitespace, remove quotes, take first line only
- If response is empty or too short (<5 chars), retry once with a simpler prompt

`generate_benchmark_pairs(n: int = TARGET_PAIRS) -> list[dict]`:
- Call `_sample_chunks_from_corpus(n)`
- For each sampled chunk, call `_generate_query()` with rotating query_type
- Return list of dicts: `{"query": str, "expected_chunk": str, "query_type": str, "source_document": str, "section_header": str}`
- Write to `PAIRS_PATH` as JSONL (one JSON object per line)
- Create `processed/benchmark/` directory if it doesn't exist

**Phase 3: Evaluate Recall@5**

`evaluate_recall(pairs: list[dict]) -> dict`:
- Embed all expected chunks using `embed_documents()` -- these form the "corpus"
- For each query: embed with `embed_query()`, compute cosine similarity against ALL chunk embeddings
- Rank by similarity (descending), check if the expected chunk is in top-K
- Use `numpy.dot()` for cosine similarity (Ollama returns L2-normalized vectors, so dot product = cosine similarity)
- Compute:
  - `recall_at_k`: fraction of queries where expected chunk is in top-K
  - `per_query_scores`: list of dicts with query, expected rank, similarity score, hit (bool)
  - `mean_reciprocal_rank`: average of 1/rank for found queries
  - `worst_performers`: bottom 10 queries by expected rank (for debugging)

`run_benchmark() -> dict`:
- Main entry point. Orchestrates all phases.
- Step 1: Generate pairs (or load existing from PAIRS_PATH if already generated -- saves LLM time on reruns)
- Step 2: Evaluate Recall@5
- Step 3: Write results to RESULTS_PATH as JSON:
  ```json
  {
    "model": "nomic-embed-text",
    "model_version": "<from get_model_version()>",
    "timestamp": "<ISO datetime>",
    "num_pairs": 60,
    "recall_at_5": 0.88,
    "pass_threshold": 0.85,
    "passed": true,
    "mean_reciprocal_rank": 0.92,
    "per_query_type_recall": {"lay_language": 0.85, "medical_terminology": 0.90, "typo_variant": 0.88},
    "worst_performers": [...]
  }
  ```
- Step 4: Print human-readable summary to stdout
- Step 5: If `passed == false`: print detailed failure report with worst performers, exit with `sys.exit(1)`
- Step 6: If `passed == true`: print success message, return results dict

**CLI entry point:**
```python
if __name__ == "__main__":
    import sys
    results = run_benchmark()
    if not results.get("passed", False):
        sys.exit(1)
```
