**Create pipeline/chunk.py** -- Content-type-aware chunking module. Key functions:

`MAX_CHUNK_TOKENS = 512` constant (using ~4 chars/token heuristic)
`MAX_CHUNK_CHARS = 2048` constant (512 * 4)

`estimate_tokens(text: str) -> int`: Return `len(text) // 4`. Conservative overestimate.

`read_section_file(filepath: Path) -> tuple[dict, str]`: Read a section Markdown file, split at `---` delimiters, parse YAML front matter with `yaml.safe_load()`, return (metadata_dict, content_str). Handle missing front matter gracefully (return empty dict).

`chunk_section(content: str, metadata: dict) -> list[ChunkRecord]`:
- Read `content_type.primary` from metadata dict. Default to "general" if empty string, missing, or None.
- Dispatch to type-specific chunker:
  - "procedure" -> `_chunk_procedure(content, metadata)`
  - "reference_table" -> `_chunk_table(content, metadata)`
  - "safety_warning" -> `_chunk_safety_warning(content, metadata)`
  - "general" (and default) -> `_chunk_general(content, metadata)`
- After chunking, set `chunk_total` on all chunk records to total count.
- Attach `warning_level` and `warning_text` from section metadata to ALL chunks (per locked decision: all warnings on all chunks in same section).

`_chunk_procedure(content: str, metadata: dict) -> list[ChunkRecord]`:
- Detect numbered steps using regex: `^(\d+[\.\)]\s|[a-z][\.\)]\s)` with `re.MULTILINE`
- Extract preamble text before first step (if any)
- Split at each step boundary
- Steps fitting within MAX_CHUNK_CHARS: emit as single chunk
- Steps exceeding MAX_CHUNK_CHARS: split at sentence boundaries using `_split_at_sentences()`
- If no numbered steps detected: fall back to `_chunk_general()`

`_chunk_table(content: str, metadata: dict) -> list[ChunkRecord]`:
- Emit entire section as a single chunk (per locked decision: tables kept whole with headers)
- If content exceeds MAX_CHUNK_CHARS (unlikely but handle it): emit as single chunk anyway with a log warning -- never split a table

`_chunk_safety_warning(content: str, metadata: dict) -> list[ChunkRecord]`:
- Emit as own retrievable chunk (per discretion recommendation: yes, safety warnings should be independently retrievable)
- If content exceeds MAX_CHUNK_CHARS: split at sentence boundaries

`_chunk_general(content: str, metadata: dict) -> list[ChunkRecord]`:
- If content fits within MAX_CHUNK_CHARS: emit as single chunk
- Otherwise: split at paragraph breaks (double newline) first
- If any paragraph exceeds MAX_CHUNK_CHARS: split at sentence boundaries
- No overlap between chunks (per discretion recommendation)

`_split_at_sentences(text: str, max_chars: int = MAX_CHUNK_CHARS) -> list[str]`:
- Split on sentence boundaries: `. `, `! `, `? ` (followed by space or newline)
- Accumulate sentences into chunks until max_chars reached
- Return list of chunk texts

`_build_chunk_record(text: str, metadata: dict, chunk_index: int) -> ChunkRecord`:
- Build ChunkMetadata from section metadata dict:
  - `source_document` = metadata["source_document"]
  - `source_title` = metadata.get("source_title", "")
  - `section_header` = metadata.get("section_heading", "")
  - `page_number` = metadata.get("page_start", 0)
  - `content_type` = metadata.get("content_type", {}).get("primary", "general") or "general"
  - `categories` = metadata.get("categories", [])
  - `source_url` = metadata.get("provenance", {}).get("source_url", "")
  - `license` = metadata.get("provenance", {}).get("license", "")
  - `distribution_statement` = metadata.get("provenance", {}).get("distribution_statement", "")
  - `verification_date` = metadata.get("processing_date", "")
  - `chunk_index` = chunk_index
  - `chunk_total` = 0 (set later by chunk_section)
  - `embedding_model` = "" (set later during embedding)
  - `embedding_model_version` = "" (set later during embedding)
  - `warning_level` = metadata.get("warning_level")
  - `warning_text` = metadata.get("warning_text")
- Return ChunkRecord(text=text.strip(), metadata=chunk_metadata)

`chunk_document(sections_dir: Path) -> list[ChunkRecord]`:
- List all .md files in sections_dir, sorted by name (order matters for section sequence)
- For each file: read_section_file(), chunk_section()
- Return flat list of all ChunkRecords for the document

Important implementation notes:
- Import yaml, re, logging, Path at module level
- Import ChunkMetadata, ChunkRecord from pipeline.models
- Use `logging.getLogger(__name__)` for module logger
- Strip leading/trailing whitespace from chunk text; skip empty chunks
- No chunk merging (per locked decision: short chunks kept as-is)
- No overlap between consecutive chunks (per discretion recommendation)

**Create pipeline/embed.py** -- Ollama nomic-embed-text embedding wrapper with batch safety and prefix convention.

Key constants:
- `BATCH_SIZE = 8` -- Safe batch size per Ollama issue #6262 (batch 16+ degrades quality)
- `DEFAULT_MODEL = "nomic-embed-text"` -- Default embedding model
- `EMBEDDING_DIM = 768` -- Expected embedding dimension

Key functions:

`get_model_version(model: str = DEFAULT_MODEL) -> str`:
- Call `ollama.show(model)` to get model info
- Extract and return the model digest/version string
- If Ollama not reachable, raise a clear error: "Ollama is not running or nomic-embed-text is not pulled. Run: ollama pull nomic-embed-text"

`embed_documents(texts: list[str], model: str = DEFAULT_MODEL) -> list[list[float]]`:
- Prepend `"search_document: "` to each text (CRITICAL: nomic-embed-text requires this prefix)
- Process in batches of BATCH_SIZE
- For each batch: call `ollama.embed(model=model, input=prefixed_batch)`
- Access embeddings from response -- try `response["embeddings"]` first, fall back to `response.embeddings` (per open question #2 in research)
- Validate each embedding has EMBEDDING_DIM dimensions
- Log progress: "Embedded batch {i}/{total} ({n} texts)"
- Return flat list of all embedding vectors

`embed_query(query: str, model: str = DEFAULT_MODEL) -> list[float]`:
- Prepend `"search_query: "` to the query (CRITICAL: different prefix for queries vs documents)
- Call `ollama.embed(model=model, input=prefixed_query)`
- Return single embedding vector

`embed_chunk_records(chunks: list[ChunkRecord], model: str = DEFAULT_MODEL) -> list[ChunkRecord]`:
- Get model version via `get_model_version(model)`
- Extract text from each ChunkRecord
- Call `embed_documents()` to get all embeddings
- Set `embedding` field on each ChunkRecord
- Set `embedding_model` and `embedding_model_version` on each ChunkRecord's metadata
- Return updated list (mutated in place for efficiency)

Important implementation notes:
- Import `ollama` at module level
- Import `ChunkRecord` from `pipeline.models`
- Use `logging.getLogger(__name__)` for module logger
- Handle `ollama.ResponseError` and `ConnectionError` with clear error messages
- Never call `ollama.embed()` directly from outside this module -- all embedding goes through these wrapper functions (prevents prefix omission per anti-pattern #1)
- The `search_document:` and `search_query:` prefixes include a trailing space -- this is required by nomic-embed-text
