Create the three remaining pipeline modules that handle LLM classification, dosage validation, and verification reporting.

**pipeline/classify.py:** LLM-based content classification via Ollama. Must include:

1. `check_ollama_ready(model: str = "llama3.1:8b") -> bool` -- Verify Ollama is running at localhost:11434 and the required model is loaded. Use `ollama.list()` to check available models. Print clear error messages with setup instructions if not ready. Follow research "Ollama Connectivity Check" pattern.

2. `CLASSIFICATION_PROMPT` -- String template for the classification prompt. Must include:
   - Instructions to classify by content type (procedure, reference_table, safety_warning, general) with clear definitions for each
   - Instructions to assign 1-3 categories from the 9-category taxonomy with definitions distinguishing `medical` from `first_aid`
   - Instructions to detect WARNING/CAUTION/NOTE with military warning level definitions
   - Instructions to extract exact warning text if present
   - `{section_text}` placeholder for the section content
   - Follow research Pattern 3 classification prompt exactly

3. `classify_section(section_text: str, model: str = "llama3.1:8b") -> SectionClassification` -- Classify a single section using Ollama structured output. Use `ollama.chat()` with `format=SectionClassification.model_json_schema()` and `options={"temperature": 0}`. Truncate input to 4000 chars to avoid variable tokenization. Parse response with `SectionClassification.model_validate_json()`. Follow research Pattern 3 exactly.

4. `classify_section_with_retry(section_text: str, model: str = "llama3.1:8b", max_retries: int = 3) -> SectionClassification` -- Wrapper that retries on Ollama errors (connection issues, malformed JSON). Log each retry. Raise after max_retries with clear error.

**pipeline/validate.py:** Dosage and measurement regex validation. Must include:

1. `DosageFlag` dataclass -- Fields: text (str), value (str), unit (str), line_number (int), reason (str), severity (str: "critical" | "review").

2. `DOSAGE_PATTERN` -- Compiled regex matching numbers followed by medical units: mg, mL, ml, cc, g, gr, grain, grains, mcg, units, tabs, caps, IU, oz, drops, tsp, tbsp, L, dL, percent, %, ppm, mmHg, bpm. Case-insensitive.

3. `OCR_CORRUPTION_PATTERNS` -- List of compiled regexes detecting common OCR errors near numbers:
   - Ambiguous 1/I/l/| adjacent to units (e.g., `Il mg`)
   - `mg` misread as `rng` or `ng`
   - Ambiguous 0/O adjacent to units
   - Implausible decimal precision (4+ decimal places before unit)
   - Implausibly large values (5+ digits before mg/mL)
   Follow research Pattern 6 exactly.

4. `validate_dosages(text: str) -> list[DosageFlag]` -- Find all dosage mentions in text. Flag implausible values (e.g., >10000 mg). Check for OCR corruption patterns. Return list of flags with line numbers.

5. `validate_section_file(file_path: Path) -> list[DosageFlag]` -- Read a section Markdown file (strip YAML front matter first), run validate_dosages on the content, return flags.

6. `generate_validation_summary(flags: list[DosageFlag]) -> str` -- Human-readable summary of all flags grouped by severity.

**pipeline/report.py:** Verification report generator. Must include:

1. `generate_report(doc_id: str, extraction_method: str, ocr_engine: str | None, total_sections: int, total_pages: int, dosage_flags: list[DosageFlag], corrections_applied: bool, section_files: list[str], reports_dir: Path) -> Path` -- Generate and write a YAML verification report per user decision "lightweight verification report per document." Report must include:
   - document (doc_id)
   - extraction_method ("born-digital" | "tesseract" | "easyocr")
   - ocr_engine (null for born-digital)
   - total_sections, total_pages
   - dosage_flags_count (int)
   - critical_flags_count (int, severity="critical" only)
   - flagged_items (list of {text, line, reason, severity})
   - sections_classified (int)
   - corrections_applied (bool)
   - processing_date (today's date)
   - status: "clean" if 0 critical flags, "needs_review" if any critical flags
   Write to `{reports_dir}/{doc_id}-report.yaml`.

2. `generate_processing_manifest(reports_dir: Path, output_path: Path) -> dict` -- Read all individual reports and generate a master `processing-manifest.yaml` with:
   - total_documents (int)
   - total_sections (int)
   - documents_needing_review (list of doc_ids with critical flags)
   - documents_clean (list of doc_ids with no critical flags)
   - processing_date

  <action>
Create the full pipeline orchestrator that runs the complete processing pipeline: extraction (from Plan 01) + classification + validation + reporting. Then run it on all 71 Tier 1 PDFs.

**pipeline/process_documents.py:** The master orchestrator. Must:

1. **Check prerequisites:**
   - Verify Ollama is running with llama3.1:8b loaded (using `check_ollama_ready()`)
   - If Ollama is not ready, print clear error with instructions: `ollama serve` and `ollama pull llama3.1:8b`
   - Verify source PDFs exist in `sources/originals/`
   - If Plan 01's extract_all.py was already run (processed/sections/ has content), SKIP extraction and go straight to classification. Otherwise, run extraction first.

2. **Accept command-line arguments:**
   - `--source-dir` (default: `sources/originals`)
   - `--manifest-dir` (default: `sources/manifests`)
   - `--output-dir` (default: `processed/sections`)
   - `--reports-dir` (default: `processed/reports`)
   - `--corrections-dir` (default: `processed/corrections`)
   - `--model` (default: `llama3.1:8b`) -- Ollama model for classification
   - `--skip-extraction` -- Skip PDF extraction, only run classification on existing sections
   - `--skip-classification` -- Skip classification, only run extraction
   - `--single` -- Process a single document by doc_id

3. **For each document (or use existing extracted sections):**
   a. If sections already exist in `processed/sections/{doc-id}/`, load them
   b. If not, run extraction pipeline (extract.py -> split.py -> clean.py -> writer.py)
   c. For each section file:
      - Read the Markdown content (strip YAML front matter)
      - Run `classify_section_with_retry()` to get classification
      - Run `validate_dosages()` on the content
      - Update the section file's YAML front matter with:
        - `content_type.primary` = classification.primary_type
        - `content_type.secondary` = classification.secondary_types
        - `categories` = classification.categories
        - `warning_level` = classification.warning_level (if any)
        - `warning_text` = classification.warning_text (if any)
   d. Apply corrections overlay if `processed/corrections/{doc-id}-corrections.yaml` exists
   e. Generate verification report via `generate_report()`

4. **Generate processing manifest:** After all documents, call `generate_processing_manifest()` to create `processed/processing-manifest.yaml`.

5. **Summary output:** Print:
   - Total documents processed
   - Total sections classified
   - Documents needing review (critical dosage flags)
   - Classification distribution (how many procedures, reference_tables, safety_warnings, general)
   - Category distribution (how many sections per category)

**If Ollama is not available:** The script should still work with `--skip-classification` to produce extracted-only section files. Classification can be run later. Log a clear message that classification was skipped.
