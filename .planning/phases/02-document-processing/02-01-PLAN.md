**pipeline/__init__.py:** Empty init file.

**pipeline/models.py:** Pydantic models used across the pipeline. Must include:

1. `ContentType(str, Enum)` -- `procedure`, `reference_table`, `safety_warning`, `general`
2. `WarningLevel(str, Enum)` -- `warning` (death/injury), `caution` (equipment), `note` (info)
3. `VALID_CATEGORIES` -- Literal type or list constant: `medical`, `water`, `shelter`, `fire`, `food`, `navigation`, `signaling`, `tools`, `first_aid`
4. `SectionClassification(BaseModel)` -- Schema for Ollama structured output (primary_type, secondary_types, categories 1-3, warning_level, warning_text, reasoning). Use `Field` with descriptions. Categories must be constrained to the 9 valid values using `Literal`.
5. `SectionMetadata(BaseModel)` -- Metadata for YAML front matter on each section file: source_document, source_title, section_order (int), section_heading, page_start (int), page_end (Optional[int]), content_type (dict with primary + secondary), categories (list[str]), warning_level (Optional[str]), warning_text (Optional[str]), extraction_method (str: "born-digital" | "tesseract" | "easyocr"), ocr_engine (Optional[str]), processing_date (str), corrections_applied (bool, default False), provenance (dict with source_url, license, distribution_statement).
6. `CorrectionEntry(BaseModel)` -- For corrections YAML overlay: section_file (str), type (str), original (str), corrected (str), reason (str).
7. `DocumentCorrections(BaseModel)` -- Wrapper: document (str), corrections_date (str), corrections_by (str), corrections (list[CorrectionEntry]).

Follow the research Pattern 3 and Pattern 5 for schema design.

**pipeline/extract.py:** Docling PDF extraction module. Must include:

1. `create_converter(ocr_backend: str = "tesseract") -> DocumentConverter` -- Creates a Docling converter with:
   - `do_ocr=True`, `do_table_structure=True`
   - `TableFormerMode.ACCURATE` for military reference tables
   - OCR backend selection (tesseract vs easyocr) via the appropriate OcrOptions class
   - `lang=["eng"]` for Tesseract, `lang=["en"]` for EasyOCR, `use_gpu=False` for EasyOCR
   - `PdfFormatOption` wrapping the pipeline options
   - `InputFormat.PDF` as the format key

2. `extract_document(pdf_path: str, ocr_backend: str = "tesseract") -> DoclingDocument` -- Converts a single PDF. Calls `converter.convert(pdf_path)` and returns `result.document`.

3. `extract_with_fallback(pdf_path: str) -> tuple[DoclingDocument, str]` -- Tries Tesseract first. If extraction fails or produces very little text (< 100 chars total), falls back to EasyOCR. Returns (document, engine_used). If both fail, raises an exception with a clear message per user decision "poor OCR quality = exclude the document."

Follow research Pattern 1 exactly for import paths and API usage.

**pipeline/clean.py:** Text cleaning module. Must include:

1. `clean_text(text: str) -> str` -- Applies all cleaning rules in order:
   - Remove residual page numbers (patterns like `\n\d{1,3}\n`, `Page \d+`, `- \d+ -`)
   - Remove repeated header/footer lines (detect lines that appear on >30% of pages)
   - Remove OCR artifacts (isolated single characters, excessive whitespace, garbled character sequences)
   - Normalize whitespace (collapse multiple blank lines to max 2, strip trailing whitespace)
   - Preserve table formatting (do NOT collapse whitespace inside Markdown table rows `|...|`)
   - Preserve WARNING/CAUTION/NOTE blocks (do NOT strip these -- they are safety-critical content)

2. `remove_page_numbers(text: str) -> str` -- Dedicated function for page number patterns.
3. `remove_ocr_artifacts(text: str) -> str` -- Remove common OCR garbage: isolated punctuation, garbled sequences of non-word characters, ligature artifacts.
4. `normalize_whitespace(text: str) -> str` -- Collapse excessive blank lines while preserving paragraph breaks and table structure.

Note: Docling already handles header/footer removal via ContentLayer.FURNITURE detection, so the cleaning here is a safety net for residual artifacts Docling misses.

**pipeline/split.py:** Section splitting module. Must include:

1. `split_into_sections(doc: DoclingDocument, source_doc_id: str) -> list[dict]` -- Walk the DoclingDocument structure using `doc.iterate_items()`. Split at section headers. Each section dict contains: heading (str), items (list), page_start (int), order (int). Use the `level` parameter from iterate_items for heading depth. Handle edge cases: untitled first section, documents with no section headers (treat entire document as one section). Follow research Pattern 2.

2. `section_to_markdown(section: dict, doc: DoclingDocument) -> str` -- Convert a section's items to Markdown. Use `isinstance(item, TextItem)` for text, `isinstance(item, TableItem)` for tables (call `item.export_to_dataframe(doc=doc)` then `.to_markdown()`). Prefix with `## {heading}\n`. Handle items that are neither TextItem nor TableItem by checking for a `.text` attribute.

3. `_get_page_number(item) -> int` -- Extract page number from a Docling item. Check `item.prov` (list of ProvenanceItem) for page numbers. Default to 1 if unavailable.

Import from `docling_core.types.doc` for `TextItem`, `TableItem`.

**pipeline/writer.py:** Markdown file writer module. Must include:

1. `write_section_file(output_dir: Path, doc_id: str, section_order: int, heading: str, markdown_content: str, metadata: SectionMetadata) -> Path` -- Write a single section as a Markdown file with YAML front matter. File naming convention: `{doc_id}_{order:03d}_{slugified_heading}.md` (e.g., `FM-21-76_001_survival-medicine.md`). Create parent directories if needed. Return the path of the written file.

2. `slugify(text: str) -> str` -- Convert heading text to filesystem-safe slug: lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, truncate to 50 chars, strip leading/trailing hyphens.

3. `apply_corrections(corrections_path: Path, sections_dir: Path) -> int` -- Read a corrections YAML file, apply text replacements to the specified section files. Return count of corrections applied. Per user decision: corrections stored separately for pipeline idempotency.

Follow research Pattern 4 for YAML front matter format and Pattern 5 for corrections overlay.

**pipeline/extract_all.py:** Main orchestration script for the extraction phase (extraction + splitting + cleaning + writing). This script does NOT do classification -- that is Plan 02.

The script must:

1. **Accept command-line arguments:**
   - `--source-dir` (default: `sources/originals`) -- Path to source PDFs
   - `--manifest-dir` (default: `sources/manifests`) -- Path to provenance manifests
   - `--output-dir` (default: `processed/sections`) -- Output directory for section files
   - `--single` (optional) -- Process a single PDF by filename for testing/debugging

2. **Discover all source PDFs:** Walk `sources/originals/{military,fema,cdc}/` subdirectories. For each PDF, find its corresponding YAML manifest in `sources/manifests/`. If no manifest exists, warn and skip (Phase 1 Plan 02 may not have run yet -- degrade gracefully by using the filename as doc_id and omitting provenance metadata).

3. **For each PDF:**
   a. Load the provenance manifest (if exists) to get document metadata
   b. Extract with Docling using `extract_with_fallback()` from extract.py
   c. Split into sections using `split_into_sections()` from split.py
   d. For each section:
      - Convert to Markdown using `section_to_markdown()`
      - Clean text using `clean_text()` from clean.py
      - Build `SectionMetadata` from manifest data + extraction info (but leave content_type and categories empty -- classification happens in Plan 02)
      - Write section file using `write_section_file()` from writer.py
   e. Log: document name, extraction method (born-digital/tesseract/easyocr), number of sections extracted, any warnings

4. **Output structure:** Create directories as needed:
   ```
   processed/sections/{doc-id}/
     {doc-id}_001_{heading-slug}.md
     {doc-id}_002_{heading-slug}.md
     ...
   ```

5. **Summary output:** At the end, print:
   - Total documents processed
   - Total sections extracted
   - Documents that used OCR (vs born-digital)
   - Any documents that failed extraction
   - Any warnings or issues

6. **Create placeholder directories:**
   - `processed/sections/.gitkeep`
   - `processed/corrections/.gitkeep`
   - `processed/reports/.gitkeep`

**After creating the script, install dependencies and run it:**

```bash
pip install -r requirements.txt
python pipeline/extract_all.py
```

Note: Docling's first run downloads model weights (~1-2 GB for layout model + TableFormer). This is expected and takes a few minutes. Subsequent runs use cached models.

If any PDFs fail extraction:
- Log the failure clearly with the document name and error
- Continue processing remaining documents (don't abort the whole batch)
- A few CDC factsheets may be very short (1-3 pages) and produce minimal sections -- that's expected

If Docling is not installable or has issues on Windows:
- Try with CPU-only PyTorch: `pip install docling --extra-index-url https://download.pytorch.org/whl/cpu`
- If `tesserocr` fails to install (common on Windows), install without OCR extras: `pip install docling pyyaml ollama pydantic` -- born-digital PDFs will still work, and most Tier 1 documents are born-digital
- Document any installation issues encountered

The extraction script MUST be runnable with `python pipeline/extract_all.py` from the project root.
