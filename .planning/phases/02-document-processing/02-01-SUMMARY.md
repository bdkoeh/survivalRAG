## Decisions Made

1. **Born-digital-first extraction strategy** -- Try extracting without OCR first since the majority of Tier 1 documents are born-digital PDFs. Only invoke OCR fallback chain when born-digital extraction produces < 100 chars. This is faster and avoids OCR engine dependency issues for born-digital documents.

2. **Doc ID logic: filename stem for civilian, designation for military** -- Military documents have well-known unique designations (FM-21-76, TC-4-02-1) that should be used as directory names. Civilian documents share generic designations (e.g., multiple "CDC Factsheet" documents), so the PDF filename stem is used instead to avoid directory collisions.

3. **OCR backend auto-detection** -- Rather than hardcoding tesserocr, the pipeline detects available OCR backends at runtime. On macOS, `ocrmac` (Apple Vision framework) is available natively without additional installation. This handles the common case where tesserocr/easyocr extras aren't installed.

4. **Section files excluded from git** -- 7,915 extracted Markdown files are pipeline output that can be regenerated in ~27 minutes. Committing them would bloat the repository. A `.gitignore` excludes `processed/sections/*/` while keeping `.gitkeep` files.
