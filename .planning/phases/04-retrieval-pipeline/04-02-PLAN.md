<objective>
Build the prompt assembly module that transforms retrieved chunks into structured prompts for LLM response generation.

Purpose: This module takes the ranked chunk list from pipeline/retrieve.py and assembles it into a structured prompt with safety warnings always injected first, context blocks with source metadata for citation, and a refusal path that short-circuits the LLM call when no chunks pass the relevance threshold. The prompt is the contract between retrieval (Phase 4) and response generation (Phase 5).

Output: pipeline/prompt.py with prompt assembly, safety warning collection/injection, and refusal handling.

**Create pipeline/prompt.py** -- Assembles retrieved chunks into structured prompts with safety warnings, source metadata, and refusal handling.

Key constants:
- `REFUSAL_MESSAGE` -- Hard canned refusal message (not LLM-generated):
  ```
  I don't have enough information in my knowledge base to answer that question reliably. Try rephrasing your query or broadening the category filter.
  ```
- `SYSTEM_PROMPT` -- System instruction for the LLM:
  ```
  You are a survival and emergency preparedness reference tool. You provide information from official US government field manuals, medical handbooks, and emergency guides.

  RULES:
  - Answer ONLY using the reference context provided below. Do not use your own knowledge.
  - If the provided context does not contain enough information to answer, say so clearly.
  - Cite your sources by document name and page number.
  - Preserve and surface all safety warnings -- never omit or downplay them.
  - Format responses as concise, actionable steps: numbered steps for procedures, bullets for lists, bold for warnings.
  - You are a reference tool, NOT a medical provider. Never diagnose conditions.
  ```

Key functions:

`collect_safety_warnings(retrieved_chunks: list[dict]) -> list[dict]`:
- Scan all retrieved chunks for warning_text in their metadata
- For each chunk where `metadata.get("warning_text")` is truthy (not None, not empty string ""):
  - Build a warning dict: `{"source_document": ..., "section_header": ..., "warning_level": ..., "warning_text": ...}`
- Deduplicate by `warning_text` content (use a set to track seen texts) -- Pitfall 6 from research
- Return list of unique warning dicts
- Order: preserve insertion order (first occurrence wins)

`assemble_prompt(query: str, retrieved_chunks: list[dict]) -> str`:
- Chunks arrive pre-sorted by RRF score (highest first) from retrieve() -- do NOT re-sort
- Build the prompt in this order:
  1. System prompt (SYSTEM_PROMPT constant)
  2. Blank line separator
  3. Safety warnings section (if any):
     - Call `collect_safety_warnings(retrieved_chunks)`
     - If warnings exist, emit:
       ```
       === SAFETY WARNINGS (MUST be included in response) ===
       WARNING [{source_document}, p.{page_number}]: {warning_text}
       ```
     - Include page_number from the chunk metadata for each warning
     - Blank line after warnings section
  4. Reference context section:
     ```
     === REFERENCE CONTEXT ===
     --- Source 1: {source_document}, Section: {section_header}, Page: {page_number} ---
     {chunk text}

     --- Source 2: ... ---
     {chunk text}
     ...
     ```
     - Number sources starting at 1
     - Each source block separated by a blank line
  5. User query section:
     ```
     === QUESTION ===
     {query}
     ```
- Return the assembled string

`build_response(query: str, retrieved_chunks: list[dict]) -> dict`:
- This is the main entry point consumed by Phase 5 (Response Generation)
- If `retrieved_chunks` is empty: return refusal result immediately
  ```python
  return {
      "status": "refused",
      "message": REFUSAL_MESSAGE,
      "prompt": None,
      "chunks": [],
      "warnings": [],
  }
  ```
- Otherwise:
  - Collect warnings via `collect_safety_warnings(retrieved_chunks)`
  - Assemble prompt via `assemble_prompt(query, retrieved_chunks)`
  - Return success result:
    ```python
    return {
        "status": "ok",
        "message": None,
        "prompt": assembled_prompt,
        "chunks": retrieved_chunks,
        "warnings": warnings,
    }
    ```
- This dict is the contract with Phase 5 -- it contains everything needed:
  - `status`: "ok" or "refused" -- Phase 5 checks this first
  - `message`: refusal text (only if refused) -- Phase 5 returns this directly
  - `prompt`: assembled prompt string (only if ok) -- Phase 5 sends this to the LLM
  - `chunks`: the retrieved chunks with metadata (for post-generation citation verification in Phase 5)
  - `warnings`: collected safety warnings (for Phase 5 to verify they appear in the response)

Important implementation notes:
- Import logging at module level
- Use `logging.getLogger(__name__)` for module logger
- No imports from pipeline.retrieve needed -- this module receives data, not fetches it
- The prompt string uses plain text markers (=== and ---) not markdown -- these are for LLM consumption, not human display
- Safety warnings appear BEFORE reference context -- safety-first principle per CLAUDE.md
- Log: "Prompt assembled: {n_chunks} chunks, {n_warnings} warnings, {len(prompt)} chars"
- Log refusal: "Query refused (no chunks passed threshold): '{query}'"
