**Create pipeline/ingest.py** -- Loads Phase 3 JSONL chunk files into ChromaDB with pre-computed embeddings and metadata.

Key constants:
- `CHROMA_PATH = "./data/chroma"` -- persistent storage directory
- `COLLECTION_NAME = "survivalrag"` -- collection name
- `BATCH_SIZE = 500` -- ChromaDB add() batch size (avoid memory issues on large ingests)

Key functions:

`get_collection(path: str = CHROMA_PATH) -> chromadb.Collection`:
- Create `chromadb.PersistentClient(path=path)`
- Call `client.get_or_create_collection()` with:
  - `name=COLLECTION_NAME`
  - `configuration={"hnsw": {"space": "cosine", "ef_construction": 200, "ef_search": 100}}`
- Return the collection
- IMPORTANT: Use `configuration` parameter (current ChromaDB API), NOT the deprecated `metadata={"hnsw:space": "cosine"}` syntax

`load_jsonl(filepath: Path) -> list[ChunkRecord]`:
- Read file line by line
- Parse each line with `json.loads()` into a dict
- Construct `ChunkRecord` from the dict using `ChunkRecord(**data)` or `ChunkRecord.model_validate(data)`
- Skip lines that fail to parse (log warning with line number and error)
- Return list of ChunkRecords

`chunk_to_chroma_id(chunk: ChunkRecord) -> str`:
- Generate a deterministic, unique ID for each chunk
- Format: `{source_document}_{page_number:03d}_{chunk_index:03d}`
- Example: `FM-21-76_045_002`

`chunk_metadata_to_dict(metadata: ChunkMetadata) -> dict`:
- Convert ChunkMetadata to a flat dict suitable for ChromaDB metadata storage
- ChromaDB metadata values must be: str, int, float, bool, or list[str/int/float]
- Map fields directly:
  - `source_document`, `source_title`, `section_header` -> str
  - `page_number`, `chunk_index`, `chunk_total` -> int
  - `content_type`, `source_url`, `license`, `distribution_statement`, `verification_date` -> str
  - `embedding_model`, `embedding_model_version` -> str
  - `categories` -> list[str] (ChromaDB >= 1.5.0 supports array metadata for $contains)
  - `warning_level` -> str or "" if None (ChromaDB does not support None values)
  - `warning_text` -> str or "" if None
- Return the dict

`ingest_chunks(chunks: list[ChunkRecord], collection: chromadb.Collection = None) -> int`:
- If collection is None, call `get_collection()` to get default collection
- Convert each chunk to: id (via chunk_to_chroma_id), document (chunk.text), embedding (chunk.embedding), metadata (via chunk_metadata_to_dict)
- Deduplicate by ID -- if multiple chunks produce the same ID, log a warning and keep the first
- Add to ChromaDB in batches of BATCH_SIZE using `collection.add(ids=..., documents=..., embeddings=..., metadatas=...)`
- Log progress: "Ingested batch {i}/{total} ({n} chunks)"
- Return total number of chunks ingested

`ingest_directory(jsonl_dir: Path, collection: chromadb.Collection = None) -> int`:
- List all .jsonl files in jsonl_dir
- For each file: load_jsonl(), then ingest_chunks()
- Log per-file: "Ingested {filename}: {n} chunks"
- Return total ingested count

`get_all_chunks_for_bm25(collection: chromadb.Collection = None) -> tuple[list[str], list[str], list[dict]]:`:
- If collection is None, call `get_collection()`
- Use `collection.get(include=["documents", "metadatas"])` to retrieve all documents
- Return tuple of (ids, documents, metadatas) for BM25 index building
- This is called at startup by the retrieval module

Important implementation notes:
- Import chromadb, json, logging, Path at module level
- Import ChunkRecord, ChunkMetadata from pipeline.models
- Use `logging.getLogger(__name__)` for module logger
- Handle empty embedding lists gracefully (skip chunks with no embedding)
- ChromaDB metadata does NOT support None values -- convert None to "" for string fields
- The `categories` field MUST be stored as a list (not a comma-separated string) for $contains filtering

**Create pipeline/retrieve.py** -- Hybrid retrieval engine with vector search, BM25 keyword search, Reciprocal Rank Fusion, cosine similarity threshold filtering, and category pre-filtering.

Key constants and configuration:
- `DEFAULT_THRESHOLD = 0.25` -- cosine similarity threshold (configurable via env var SURVIVALRAG_RELEVANCE_THRESHOLD)
- `DEFAULT_MAX_RESULTS = 5` -- maximum chunks to return (configurable via env var SURVIVALRAG_MAX_CHUNKS)
- `RRF_K = 60` -- RRF fusion constant (standard value from Cormack et al. 2009)

Module-level state:
- `_bm25_index: Optional[bm25s.BM25] = None` -- in-memory BM25 index, built at startup
- `_bm25_ids: list[str] = []` -- chunk IDs corresponding to BM25 index positions
- `_bm25_categories: dict[str, list[str]] = {}` -- mapping from chunk ID to categories, for post-hoc BM25 category filtering
- `_collection: Optional[chromadb.Collection] = None` -- cached ChromaDB collection reference

Key functions:

`init(chroma_path: str = None) -> None`:
- Initialize the retrieval engine: connect to ChromaDB and build BM25 index
- Get the ChromaDB collection via `ingest.get_collection(path=chroma_path)` if chroma_path provided, else `ingest.get_collection()`
- Store in module-level `_collection`
- Call `ingest.get_all_chunks_for_bm25(collection=_collection)` to get (ids, docs, metas)
- Build BM25 index: call `build_bm25_index(ids, docs, metas)`
- Log: "Retrieval engine initialized: {n} chunks in index"

`build_bm25_index(ids: list[str], documents: list[str], metadatas: list[dict]) -> None`:
- Set module-level `_bm25_index`, `_bm25_ids`, `_bm25_categories`
- `_bm25_ids = ids`
- `_bm25_categories = {id: meta.get("categories", []) for id, meta in zip(ids, metadatas)}`
- Tokenize documents: `corpus_tokens = bm25s.tokenize(documents)`
- Create and index: `_bm25_index = bm25s.BM25()` then `_bm25_index.index(corpus_tokens)`
- Log: "BM25 index built: {len(ids)} documents"

`_build_category_filter(categories: Optional[list[str]]) -> Optional[dict]`:
- If categories is None or empty: return None (search all)
- If single category: return `{"categories": {"$contains": categories[0]}}`
- If multiple categories: return `{"$or": [{"categories": {"$contains": c}} for c in categories]}`

`_vector_search(query_embedding: list[float], n_results: int, where_filter: Optional[dict]) -> list[dict]`:
- Call `_collection.query(query_embeddings=[query_embedding], n_results=n_results, where=where_filter, include=["documents", "metadatas", "distances"])`
- Convert ChromaDB column-major results to list of dicts:
  ```python
  results = []
  for doc_id, distance, doc, meta in zip(
      raw["ids"][0], raw["distances"][0],
      raw["documents"][0], raw["metadatas"][0]
  ):
      results.append({
          "id": doc_id,
          "similarity": 1 - distance,  # CRITICAL: convert distance to similarity
          "text": doc,
          "metadata": meta,
      })
  ```
- Return list of result dicts

`_bm25_search(query: str, n_results: int, categories: Optional[list[str]]) -> list[dict]`:
- Tokenize query: `query_tokens = bm25s.tokenize(query)`
- Retrieve: `results, scores = _bm25_index.retrieve(query_tokens, k=min(n_results, len(_bm25_ids)))`
- `results` shape is (1, k) containing indices into the corpus; `scores` shape is (1, k) containing BM25 scores
- Map indices back to chunk IDs using `_bm25_ids`
- CRITICAL: Apply category filtering post-hoc BEFORE returning results:
  - If categories is not None and not empty:
    - Filter results to only include chunks whose `_bm25_categories[chunk_id]` contains at least one of the requested categories
  - This prevents BM25 results from wrong categories leaking through RRF fusion (Pitfall 2 from research)
- Return list of dicts: `[{"id": chunk_id, "bm25_score": score}, ...]`
- Filter out results with score <= 0 (no BM25 match)

`reciprocal_rank_fusion(vector_results: list[dict], bm25_results: list[dict], k: int = RRF_K) -> list[dict]`:
- Combine two ranked lists using RRF: `score(d) = sum(1 / (k + rank))` across lists where doc appears
- For each result in vector_results (already ranked by similarity):
  - `rrf_scores[id] += 1 / (k + rank + 1)` (rank is 0-based, so add 1)
  - Store text, metadata, similarity from vector results
- For each result in bm25_results (already ranked by BM25 score):
  - `rrf_scores[id] += 1 / (k + rank + 1)`
  - If this ID was not in vector results, it lacks text/metadata -- these BM25-only results are excluded from final output (they have no ChromaDB metadata to build a response from)
- Sort by RRF score descending
- Return list of dicts with keys: id, rrf_score, similarity, text, metadata
- Only return results that have full metadata (present in vector results)

`retrieve(query: str, categories: Optional[list[str]] = None, n_results: int = None, threshold: float = None) -> list[dict]`:
- This is the main entry point for retrieval
- Read defaults from env vars if not provided:
  - `n_results`: `int(os.environ.get("SURVIVALRAG_MAX_CHUNKS", DEFAULT_MAX_RESULTS))`
  - `threshold`: `float(os.environ.get("SURVIVALRAG_RELEVANCE_THRESHOLD", DEFAULT_THRESHOLD))`
- Check that `_collection` and `_bm25_index` are initialized; if not, raise RuntimeError("Retrieval engine not initialized. Call init() first.")
- Step 1: Embed query via `embed_query(query)` from pipeline.embed
- Step 2: Build category filter via `_build_category_filter(categories)`
- Step 3: Run vector search -- `_vector_search(query_embedding, n_results * 2, where_filter)` (over-fetch for fusion)
- Step 4: Run BM25 search -- `_bm25_search(query, n_results * 2, categories)` (category-filtered)
- Step 5: Fuse via RRF -- `reciprocal_rank_fusion(vector_results, bm25_results)`
- Step 6: Threshold filter -- keep only results where `similarity >= threshold`
- Step 7: Return top `n_results` from filtered list, or empty list if none pass threshold
- Log: "Query: '{query}' | Categories: {categories} | Vector: {n_vector} | BM25: {n_bm25} | Fused: {n_fused} | Passed threshold: {n_passed}"

Important implementation notes:
- Import os, logging, bm25s, chromadb at module level
- Import embed_query from pipeline.embed
- Import get_collection, get_all_chunks_for_bm25 from pipeline.ingest
- Use `logging.getLogger(__name__)` for module logger
- typing imports: Optional from typing
- numpy is NOT needed -- RRF uses only basic Python arithmetic
- Handle edge cases:
  - Empty collection (no chunks indexed): return empty list
  - BM25 index not built: raise RuntimeError
  - Query returns 0 vector results: RRF still works with BM25 results only (but those lack metadata, so return empty)
  - All results below threshold: return empty list (triggers refusal in caller)
