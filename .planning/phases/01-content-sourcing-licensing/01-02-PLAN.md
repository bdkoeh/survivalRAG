**YAML Manifest Schema (use for EVERY manifest):**
```yaml
# Provenance Manifest for [Document Title]
# Schema version: 1.0

document:
  title: "[Full title]"
  designation: "[FM/TC/etc number or publication name]"
  edition_date: "[YYYY-MM-DD or YYYY]"
  pages: [number or "unknown"]
  file_name: "[exact filename in originals/]"
  file_sha256: "[from sources/checksums.sha256]"

source:
  primary_url: "[URL used for download]"
  official_listing_url: "[.gov/.mil URL where publication is listed]"
  fallback_urls:
    - "[alternate URL]"
  publisher: "[Department of the Army / FEMA / CDC]"
  country: "United States"

licensing:
  license_type: "[US Government Work - Public Domain]"
  distribution_statement: "[exact statement text]"
  distribution_statement_source: "[where statement was found]"
  copyright_status: "No copyright (17 U.S.C. 105)"
  verification_date: "[date verified]"
  verification_method: "[how distribution statement was verified]"
  verified_by: "automated"

content:
  categories: [list of applicable categories from: survival, shelter, water, food, navigation, fire, tools, first_aid, medical, signaling, hygiene, sanitation, preparedness, emergency_response]
  tier: 1
  content_type: "[field_manual / guide / factsheet / reference]"
  language: "en"

processing:
  notes: "[born-digital or scanned, text quality notes]"
  ocr_needed: [true/false]
  download_date: "[actual download date]"
  download_method: "curl"

superseded_by: "[newer publication if applicable, or null]"
supersedes: "[older publication if applicable, or null]"
```

Create a Python validation script and run it to verify the complete provenance chain.

**validate-manifests.py** must check:

1. **Schema completeness** -- Every YAML manifest has all required top-level keys: `document`, `source`, `licensing`, `content`, `processing`
2. **Required fields within sections:**
   - `document.title`, `document.file_name`, `document.file_sha256`
   - `source.primary_url`, `source.publisher`
   - `licensing.license_type`, `licensing.distribution_statement`, `licensing.verification_date`, `licensing.verification_method`
   - `content.categories` (non-empty list), `content.tier` (must be 1)
   - `processing.download_date`
3. **Cross-reference checks:**
   - Every `document.file_name` must correspond to an existing file in `sources/originals/` (in the correct subdirectory)
   - Every `document.file_sha256` must match a line in `sources/checksums.sha256`
   - The SHA-256 in the manifest must match the actual SHA-256 of the file on disk
4. **Licensing verification:**
   - Military documents must have `distribution_statement` containing "Approved for public release" or "Distribution Statement A"
   - FEMA/CDC documents must have `license_type` containing "US Government Work" or "Public Domain"
5. **Orphan check:**
   - Every PDF in `sources/originals/` must have a corresponding manifest
   - Every manifest must have a corresponding PDF
6. **Excluded document cross-check:**
   - Verify no manifest exists for FM 3-05.70, ST 31-91B, or any document mentioned in EXCLUDED.md

**Script output:** Print a clear pass/fail per check, per file. Exit 0 if all pass, exit 1 if any fail.

**Usage:** `python sources/scripts/validate-manifests.py`

The script should use only Python standard library (yaml is in stdlib as of Python 3.x -- use `yaml.safe_load`). Wait -- PyYAML is NOT in the Python standard library. Use `pip install pyyaml` first, or better yet, parse YAML with a simple approach. Actually, use PyYAML since it's the standard approach. The validate script should note at the top that it requires `pip install pyyaml` if not already installed.

After creating the script, run it: `python sources/scripts/validate-manifests.py`

Fix any validation errors found. Re-run until all checks pass.
