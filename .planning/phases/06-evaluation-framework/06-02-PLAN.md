<objective>
Build the evaluation runner that measures retrieval recall, hallucination refusal, citation faithfulness, and safety warning surfacing against the golden dataset from Plan 01.

Purpose: The evaluation framework is the quality gate proving the pipeline works, not just demos well. It transforms subjective "seems to work" into quantitative metrics with pass/fail thresholds. Without it, regressions in retrieval quality, citation accuracy, or refusal behavior go undetected.

Output: `pipeline/evaluate.py` -- a single module that runs all four evaluation dimensions, prints a terminal summary table, writes detailed JSON results, and exits non-zero on threshold failures.
</objective>

**Dimension 1 -- Retrieval Recall (EVAL-02):**
```python
def evaluate_retrieval(entries: list[dict]) -> list[dict]:
```
- For each golden query: call `retrieve.retrieve(query, categories=[category] if category else None)`
- Score = proportion of expected_chunk_ids found in retrieved result IDs
- Return per-query result dicts with: query, query_type, category, dimension="retrieval", score, expected, actual, hits, misses
- Print progress: `Evaluating retrieval: {i}/{n}...` (per Pitfall 5)

**Dimension 2 -- Refusal / Hallucination (EVAL-03):**
```python
def evaluate_refusal(entries: list[dict]) -> list[dict]:
```
- For each refusal query: call `gen.answer(query)`
- Per CONTEXT.md discretion recommendation: check `result["status"] == "refused"` (status check, NOT exact message string matching)
- Score: 1.0 if refused, 0.0 if not
- Return per-query result dicts with: query, query_type, dimension="refusal", score, passed, status, response_preview (first 100 chars if not refused, for debugging)
- Print progress: `Evaluating refusal: {i}/{n}...`

**Dimension 3 -- Citation Faithfulness (EVAL-04):**
```python
def evaluate_citation_faithfulness(entries: list[dict]) -> list[dict]:
```
- For each golden query: call `gen.answer(query, mode="full")` (per Pitfall 3: only test "full" mode, NOT ultra)
- Skip entries where `result["status"] == "refused"` (refusal = not a citation failure)
- Use `result["verification"]` dict directly -- it already has citations_found, citations_verified, citations_failed
- Score = citations_verified / citations_found if citations_found > 0, else 1.0 (no citations is not a failure per research)
- Return per-query result dicts with: query, dimension="citation_faithfulness", score, citations_found, citations_verified, citations_failed, details
- Print progress: `Evaluating citation faithfulness: {i}/{n}...`

**Dimension 4 -- Safety Warning Surfacing (EVAL-05):**
```python
def evaluate_safety_warnings(entries: list[dict]) -> list[dict]:
```
- Filter to entries where `safety_critical == True` only
- For each: call `retrieve.retrieve(query)` and then `prompt.collect_safety_warnings(retrieved_chunks)`
- Score: 1.0 if warnings list is non-empty, 0.0 if empty
- Return per-query result dicts with: query, dimension="safety_warnings", score, passed, warnings_found, warning_texts (first 80 chars each)
- Print progress: `Evaluating safety warnings: {i}/{n}...`

**Suite selection (per discretion recommendation: support --suite flag):**
Support `--suite` argument via argparse:
- `--suite all` (default): Run all 4 dimensions
- `--suite retrieval`: Run only retrieval recall + safety warnings (no LLM needed, fast)
- `--suite refusal`: Run only refusal testing
- `--suite citation`: Run only citation faithfulness
- `--suite safety`: Run only safety warning surfacing

This enables fast iteration during development per Pitfall 5 (retrieval-only takes seconds, full pipeline takes minutes).

**Main block:**
```python
def main():
    """Run evaluation suite."""
```
1. Parse `--suite` argument (default: "all")
2. Call `init()` -- fail fast if Ollama not running
3. Load datasets from GOLDEN_PATH and REFUSAL_PATH
4. Run pre-flight `validate_chunk_ids()` on golden entries -- log warning if missing IDs but continue

6. Aggregate results per dimension
7. Compute overall_passed: ALL thresholds met
8. Print terminal summary table
9. If failures exist, print FAILURES detail section
10. Write JSON results
11. `sys.exit(0 if overall_passed else 1)` per CONTEXT.md non-zero exit code decision
