Add the streaming chat handler and status update logic to web.py:

**1. Streaming chat response function:**
```python
def chat_respond(message, history, selected_categories, mode):
    """Stream response tokens to Gradio chatbot with citation links and warnings."""
```
Implementation:
- Guard: if `not message.strip()`, return without yielding
- Add user message to history: `history = history + [{"role": "user", "content": message}]`
- Call `gen.answer_stream(query_text=message, categories=selected_categories or None, mode=mode)`
- Handle refusal path (`status == "refused"`): yield the refusal message as complete assistant response
- Handle ok path: stream tokens, accumulating `partial` response text
  - On each token, yield updated history with `{"role": "assistant", "content": partial}`
- After streaming completes (for ok path):
  - Run `citations_to_links(partial)` on the completed response to convert citations to clickable PDF links
  - Prepend safety warnings if available: call `retrieve.retrieve(message, categories=selected_categories or None)` and then `from pipeline.prompt import collect_safety_warnings; collect_safety_warnings(chunks)` to get warnings. Use `format_warnings_html(warnings)` to create warning blocks. Prepend them to the response.
  - Yield final history with the fully post-processed response

**IMPORTANT streaming detail:** During streaming, yield raw tokens (no link conversion -- citations may be incomplete mid-token). After the full response is collected, do ONE final yield with the post-processed version (citations converted to links, warnings prepended). This avoids broken links from partial citation text during streaming.

**2. Wire Gradio events:**
