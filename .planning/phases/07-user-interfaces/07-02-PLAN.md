**2. Pipeline initialization helper:**

```python
def _init_pipeline():

```
- Call `retrieve.init(chroma_path="./data/chroma")`
- Call `gen.init()`
- Wrap in try/except:
  - ConnectionError -> click.echo("Error: Ollama is not running. Start with: ollama serve", err=True); sys.exit(1)
  - RuntimeError -> click.echo(f"Error: {e}", err=True); sys.exit(1)
  - Exception -> click.echo(f"Startup error: {e}", err=True); sys.exit(1)

**3. Click group (entry point):**
```python
@click.group(invoke_without_command=True)
@click.pass_context
def cli(ctx):
    """SurvivalRAG: Offline survival and emergency preparedness knowledge base."""
    if ctx.invoked_subcommand is None:
        repl()
```
When no subcommand is given, drop into REPL mode.

**4. Ask subcommand:**
```python
@cli.command()
@click.argument("query")
@click.option("--category", "-c", default=None, help="Comma-separated categories: medical,water,shelter,fire,food,navigation,signaling,tools,first_aid")
@click.option("--mode", "-m", "response_mode", default="full", type=click.Choice(["full", "compact", "ultra"]), help="Response mode (default: full)")
def ask(query, category, response_mode):
    """Ask a survival or emergency preparedness question."""
```
Implementation:
- Parse categories: `categories = [c.strip() for c in category.split(",")] if category else None`
- Call `_init_pipeline()`
- Call `gen.answer(query_text=query, categories=categories, mode=response_mode)` -- use non-streaming `answer()` for single-shot (collect full response for Rich rendering)
- If `result["status"] == "refused"`: display refusal message and return
- If `result.get("warnings")`: display each warning as a Rich Panel (see Task 2 for formatting)
- Display main response as Rich Markdown
- Exit cleanly

**Why use `answer()` not `answer_stream()` for single-shot:** Rich Markdown rendering needs the complete response text. Streaming token-by-token to the terminal with Rich is unnecessarily complex for single-shot mode and would not look good (tokens appear raw, then get re-rendered). The user sees the complete formatted output after a brief wait. For REPL mode, use `answer_stream()` with raw token printing for responsiveness, then format the complete response after.

**5. REPL function:**
```python
def repl():
    """Interactive REPL for querying the knowledge base."""
```
Implementation:
- Print startup banner: "SurvivalRAG v1.0" and "Reference tool only -- not medical advice." (per CONTEXT.md)
- Call `_init_pipeline()`
- Print ready line: f"Ready. Model: {gen._model} | Chunks: {retrieve._collection.count():,}"
- Print help hint: "Type a question, or 'quit' to exit. Use --help for flags.\n"
- Enter input loop:
  - Prompt: use Rich Console input or plain `input(">> ")` wrapped in try/except (EOFError, KeyboardInterrupt -> clean exit)
  - Skip empty input
  - Handle quit commands: "quit", "exit", "q"
  - Handle mode prefix shortcuts (keeping compatibility with ask.py patterns):
    - `/compact <query>` -> mode="compact"
    - `/ultra <query>` -> mode="ultra"
    - `/full <query>` -> mode="full"
    - No prefix -> mode="full"
  - Handle category prefix: `/category medical,water <query>` or `/cat medical <query>`
  - Stream response: call `gen.answer_stream(query_text=query, categories=categories, mode=mode)`
  - For streaming in REPL: print tokens as they arrive (`print(token, end="", flush=True)`), collect full response, then after streaming completes, optionally re-render with Rich formatting for warnings
  - Actually: for REPL simplicity, use `gen.answer()` (non-streaming) and display with Rich, matching single-shot behavior. The existing ask.py streams raw text but this CLI upgrades to Rich-formatted output. The brief wait for full response is acceptable in REPL mode given the formatted output quality.
  - After response: print blank line separator

**6. Main block:**
```python
if __name__ == "__main__":
    cli()
```

**Design decisions:**
- Use `gen.answer()` (not `answer_stream()`) for BOTH single-shot and REPL to get the full response dict including warnings and verification. This enables proper Rich formatting of the complete response.
- REPL disclaimer is a short one-liner per CONTEXT.md. No disclaimer in single-shot mode (pipe-friendly).
- Category and mode flags use standard Click conventions (`--category`, `--mode`).
- REPL supports `/compact`, `/ultra`, `/full` prefix shortcuts for quick mode switching.

**3. Response display function:**
```python
def display_response(result: dict):
    """Render a pipeline response dict with Rich formatting.

    Displays safety warnings as colored panels, then the main response
    as rendered markdown. Handles refusal responses gracefully.
    """
```
Implementation:
- If `result["status"] == "refused"`:
  - `console.print(Panel(result["response"], title="No Results", border_style="yellow"))`
  - Return early
- Display safety warnings FIRST (safety-first principle from CLAUDE.md):
  - For each warning in `result.get("warnings", [])`:
    - Create a Rich Panel with:
      - Content: `[bold]{warning['warning_text']}[/bold]`
      - Title: `WARNING -- {warning['source_document']}, p.{warning['page_number']}`
      - border_style: `"red"` for warning_level "danger" or "caution", `"yellow"` for "warning" or default
    - `console.print(panel)`
- Display main response as Rich Markdown:
  - `console.print(Markdown(result["response"]))`
- If citation verification failed (result.get("verification", {}).get("passed") is False):
  - `console.print("[dim italic]Note: Some citations could not be verified against source documents.[/dim italic]")`

**4. Update ask() command to use display_response():**
Replace any raw print() calls in the ask() function with `display_response(result)`. The `gen.answer()` return dict has all the data needed: response, status, warnings, verification.

**5. Update repl() to use display_response():**
In the REPL loop, after calling `gen.answer()`, pass the result to `display_response(result)`.

**6. Styled REPL startup banner:**
Replace plain print() calls in repl() startup with Rich formatting:
```python
console.print("[bold green]SurvivalRAG v1.0[/bold green]")
console.print("[dim]Reference tool only -- not medical advice.[/dim]")
# After init:
console.print(f"[green]Ready.[/green] Model: {gen._model} | Chunks: {retrieve._collection.count():,}")

```

**7. REPL prompt styling:**
Use Rich for the prompt:

**8. Error display in REPL:**
On exceptions during query, display with Rich:


**Design decisions (Claude's discretion from CONTEXT.md):**
- Rich is used for ALL terminal output (per CONTEXT.md locked decision: "Rich markdown with color")
- Warning panels use red border for danger/caution, yellow for standard warnings
- Citation verification failure shown as dim italic note (unobtrusive but visible)
- REPL prompt uses green `>>` to match terminal aesthetic
- Single-shot mode has NO decorative output beyond the response itself -- suitable for piping (`python cli.py ask "query" | less`)
